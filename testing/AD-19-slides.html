Test-Driven Development

El ciclo Red · Green · Refactor

1

🔴 Red

Escribe la prueba.
Córrela. Debe fallar.

→
2

🟢 Green

Código mínimo
que la haga pasar.

→
3

🔵 Refactor

Mejora el código.
Las pruebas siguen verdes.

→
↩

Repeat

Nueva función,
nuevo ciclo.

Regla clave

Si la prueba pasa antes de escribir código, la prueba está mal.

Regresión

Una prueba que antes pasaba y ahora falla = nuevo código rompió algo existente.

Producción

Integración Continua

Cada commit dispara las pruebas. Si fallan, el código no avanza. El equipo lo sabe en minutos.

Entrega Continua

Si las pruebas pasan, el código está siempre listo para producción. No hay "día de deploy".
